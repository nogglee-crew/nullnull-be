# Room Lifecycle

SRS 원문 기준 핵심 객체는 `Room`, `Participant`, `RoomResult`다. 이 문서는 백엔드가 가장 먼저 고정해야 하는 방 라이프사이클만 분리해 요약한다.

## Main States

- `DRAFT`: 방 생성 직후, 입력 수집 전
- `COLLECTING`: 참여자 입력 수집 중
- `READY`: 계산 가능한 상태
- `CONFIRMED`: 방장이 결과 확정
- `CLOSED`: 만료 또는 종료

상세 상태명은 구현 전에 확정해도 되지만, 전이 규칙은 명시적으로 유지한다.

## Core Requirements

- 방은 고유 slug로 식별된다.
- 방장은 익명 세션 토큰으로 권한을 증명한다.
- 참여자는 방 단위 토큰으로 자기 입력만 수정할 수 있다.
- 확정은 단일 결과 스냅샷을 가리켜야 한다.
- 만료 정책은 시간 기반으로 자동 처리될 수 있다.

## Write Paths

- 방 생성
- 방 설정 수정
- 참여자 입장 또는 재입장
- 불가 시간 제출
- 위치 제출
- 결과 확정
- 방 종료 또는 만료 처리

## Read Paths

- 방 기본 정보 조회
- 참여자 요약 조회
- 최신 계산 결과 조회
- 확정 결과 조회
