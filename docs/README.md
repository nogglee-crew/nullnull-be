# Docs

이 디렉터리는 `nullnull` 백엔드의 요구사항, 작업 기록, 협업 규칙을 관리한다.
문서는 구현보다 먼저 확인하고, 코드 변경으로 기준이 바뀌면 함께 갱신한다.

## Read Guide

- 기능 요구사항이나 도메인 규칙을 확인할 때는 `specs/README.md`를 먼저 읽는다.
- 작업 방식, 브랜치, 커밋, PR 규칙은 `conventions/`를 따른다.
- 복잡한 기능, 큰 리팩터링, 결정이 필요한 작업은 `exec-plans/`에 계획과 변경 이력을 남긴다.
- 인증, 토큰, 개인정보, 위치 정보, 외부 API 키를 다루는 작업은 `SECURITY.md`를 함께 확인한다.

## Source Of Truth

- `specs/`: 기능 요구사항, 도메인 계약, 설계 원칙.
- `exec-plans/`: 진행 중 작업 계획, 완료 기록, 기술 부채.
- `conventions/`: 작업 원칙, 브랜치, 커밋, PR 작성 규칙.
- `SECURITY.md`: 보안 기준.

## Update Rules

- 코드 구조를 바꾸면 설계 문서를 같이 갱신한다.
- 계약을 바꾸면 spec 문서를 갱신한다.
- 복잡한 작업은 실행 계획 없이 바로 구현하지 않는다.
- 문서와 코드가 충돌하면 충돌 사실을 실행 계획에 먼저 기록하고, 기준 문서를 갱신한 뒤 구현을 맞춘다.
